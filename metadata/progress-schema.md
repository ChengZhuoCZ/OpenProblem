# Progress File Schema

Each problem directory should contain a `progress.md` file.

Required sections:

- `Current Status`: short status label and one-line summary.
- `What Is Already Organized`: facts or formulations already captured in `problem.md`.
- `Open Clarifications`: assumptions, definitions, or scope questions that must be resolved.
- `Next Actions`: concrete work items.
- `Work Log`: dated notes in reverse chronological order.

Progress files should track research state, not duplicate the full problem statement.
